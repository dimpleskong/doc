# 自动化测试-接口依赖

# 一.常见依赖方式

# 1.预处理参数 ---配置文件读取

  • 优点：通用参数，参数为固定 （用例中/框架中都可以使用）一般用于环境变量
  • 缺点 ：不便捷

# 2.获取依赖用例实际参数

  • 分别执行接口，存储需要依赖的数据（文档/redis/内存），等到依赖接口要使用的时候再去获取

  • 缺点 ：如果前接口的参数失败，则依赖接口必定也是失败 在并发执行中,会存在设计上的缺陷（调用用例的时候串联的执行）

  • 优点： 成本相对低

# 3.调用接口实时产生数据

  • 缺点： 成本相对较大，一般是使用造数据的接口或者平台去生成前置依赖
  • 优点： 该依赖可以更大程度解耦用例和用例之间的依赖，稳定性/准确性会更好
